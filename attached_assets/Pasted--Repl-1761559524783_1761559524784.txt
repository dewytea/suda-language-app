=== Writing Phase 2: GPT-4 첨삭 시스템 ===

**목표:**
사용자가 작성한 글을 제출하면 GPT-4가 상세한 첨삭을 제공하고, 
결과를 "My Writings" 페이지에서 확인할 수 있도록 구현

**주요 기능:**

1. **제출 버튼 추가**
   - Phase 1의 작성 페이지에 "제출하기" 버튼
   - 클릭 시 자동으로 GPT-4 첨삭 시작
   - 로딩 인디케이터 표시

2. **GPT-4 첨삭 API 연동**
   - OpenAI GPT-4o 사용
   - 다음 항목을 분석:
     * 문법 오류 (Grammar Errors)
     * 개선 제안 (Improvement Suggestions)
     * 전체 교정본 (Corrected Version)
     * 강점 (Strengths)
     * 개선점 (Areas for Improvement)
     * 종합 점수 (0-100)

3. **첨삭 결과 페이지**
   - 자동으로 결과 페이지로 리다이렉트
   - 원본 글과 첨삭 결과 나란히 표시
   - 각 오류에 마우스 오버 시 설명 표시
   - 점수를 시각적으로 표시 (진행바, 원형 게이지 등)

4. **My Writings 페이지**
   - 제출한 모든 글 목록
   - 각 글의 정보:
     * 제목 (토픽)
     * 제출 날짜
     * 점수
     * 오류 개수
   - 클릭하면 상세 첨삭 결과 보기
   - 날짜순/점수순 정렬 기능

5. **데이터베이스 스키마**
```sql
   CREATE TABLE writing_submissions (
     id UUID PRIMARY KEY,
     user_id UUID REFERENCES profiles(id),
     topic TEXT NOT NULL,
     difficulty TEXT NOT NULL,
     category TEXT NOT NULL,
     content TEXT NOT NULL,
     word_count INTEGER,
     score INTEGER,
     grammar_errors JSONB,
     suggestions JSONB,
     corrected_content TEXT,
     strengths TEXT[],
     improvements TEXT[],
     created_at TIMESTAMP DEFAULT NOW()
   );
```

```

   학생이 작성한 글: {content}
   토픽: {topic}
   난이도: {difficulty}
   
   다음 형식으로 JSON 응답해주세요:
   {
     "score": 0-100 점수,
     "grammar_errors": [
       {
         "error": "틀린 문장",
         "correction": "올바른 문장",
         "explanation": "설명"
       }
     ],
     "suggestions": [
       {
         "original": "개선 전 문장",
         "improved": "개선 후 문장",
         "reason": "개선 이유",
         "example": "예시"
       }
     ],
     "corrected_content": "완전히 교정된 전체 글",
     "strengths": ["강점1", "강점2"],
     "improvements": ["개선점1", "개선점2"]
   }
```

7. **UI 컴포넌트**
   - WritingSubmitButton: 제출 버튼
   - FeedbackPage: 첨삭 결과 페이지
   - MyWritingsPage: 제출 글 목록
   - ErrorHighlight: 오류 강조 컴포넌트
   - ScoreDisplay: 점수 표시 컴포넌트
